# TWRP Recovery 音频 Bring-up 最终状态总结

## 设备信息
- 设备：OPD2413 / ossi
- 平台：Qualcomm（lpass-cdc / spf_core_platform）
- 环境：TWRP Recovery（Android 12/14 混合编译）
- 声卡：sun-mtp-snd-card
- 扬声器：AW882xx Smart PA（aw882xx_smartpa.4-0034 等 6 个 I2C 设备）

---

## 各阶段状态

### 阶段1：起卡 — 自动注册声卡 ✅
- 旧 recovery_a.img（3月10日）不含源树改动
- magiskboot 解包 ramdisk → 注入新版 init + twrp-audio-autoload.sh + vendor.audio-recovery-services.rc
- init trigger 从 twrp.audio.autoload.requested=true 改为 twrp.modules.loaded=true
- 结果：冷启动自动起卡成功

### 阶段2：模块缺口 — BTFM/rfkill 依赖链 ✅
- hdmi_dlkm 已加载，但 sound 设备 deferred
- 逐步补模块：aw882xx → machine_dlkm → btpower（缺 rfkill_* 符号）
- rfkill.ko 存在于 system_dlkm/lib/modules/
- 修复：脚本加入 system_dlkm 路径 + BTFM 依赖链完整加载
- 结果：冷启动后所有关键模块自动加载，devices_deferred 清空

### 阶段3：播放工具 — 厂商 FTM 工具探索 ✅
- 发现设备端厂商工具：/odm/bin/tinyplay_ftm、/vendor/bin/agmplay、/vendor/bin/mm-audio-ftm、/vendor/bin/cplay
- tinyplay_ftm + TDM-LPAIF-RX-SECONDARY 播放厂测 wav 返回 RC:0
- 问题：实际无声

### 阶段4：AGM/HAL 控制面缺失 ⚠️
- tinyplay_ftm 真实错误：device_config not found / Invalid mixer control
- audiohalservice.qti 被 audioeffecthal.qti（VINTF 无声明）反复拉崩，退回 audiohal-default
- 结论：所有播放工具卡在 AGM 无法找到 backend mixer control

### 阶段5：DT/DTBO 修补 — WSA/SWR 节点 ✅
- sound 节点 upd_backends_used = "wsa"，但 wsa-macro/wsa2-macro 是 disabled
- 三轮 dtbo patch：
  1. wsa/wsa2 macro status → "okay"
  2. 4 个 pinctrl 节点 → "okay"
  3. qcom,wsa-max-devs 从 0 改为 3
- 结果：WSA 平台设备首次实例化
- 剩余：SWR controller version mismatch

### 阶段6：tinymix + tinyplay PCM 直推验证 ⚠️
- 从 stock 导出 tinymix、tinyplay 到 recovery
- 手工用 tinymix 写入完整 WSA speaker 路由
- 真 tinyplay 对 card0:device16/24 做 PCM 直推
- 结果：所有格式均未能将音频推进到 AW882xx speaker DAPM

### 阶段7：Stock 对照 — 精确 KV 参数抓取 ✅
- Stock speaker 走 CODEC_DMA-LPAIF_WSA-RX-0、pcm_id_rx:100
- 从 usecaseKvManager.xml 提取：device_kv=0xA2000001 等
- Stock 能到 aw882xx_startup/hw_params，卡在 graph_open -19
- Recovery 死在 IAGM/default 缺失

### 阶段8：VINTF manifest 音频服务声明修补 ⚠️（当前前沿）
- 补入 manifest_audiocorehal_default.xml 等 4 个缺失 VINTF fragment
- IConfig/default、IModule/*、IFactory/default 门槛消失
- IAGM/default 仍然注册失败
- 根因：vendor 音频 userspace 与 recovery system libbinder_ndk 存在 ABI 裂口

## 当前最硬阻塞
Recovery 的 vendor 音频 userspace（AGM/PAL/audiohalservice）和 recovery system libraries（libbinder_ndk）存在 ABI 版本不匹配，导致 libagmipcservice.so 无法完成 registerService，IAGM 起不来，AGM graph/card100 不可用。
