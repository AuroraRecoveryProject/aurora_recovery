# 调试底层库崩溃堆栈

## 背景

Android Recovery 环境中没有 debuggerd / tombstone 服务，Runner 崩溃时只能靠自带的信号处理器输出原始 PC 地址。

```c
static void signal_handler(int sig) {
  std::cerr << "\n=== CRASH: Signal " << sig << " (";
  if (sig == SIGTRAP) std::cerr << "SIGTRAP";
  else if (sig == SIGABRT) std::cerr << "SIGABRT";
  else if (sig == SIGSEGV) std::cerr << "SIGSEGV";
  else std::cerr << "unknown";
  std::cerr << ") ===" << std::endl;
  
  // Print relevant memory maps
  std::cerr << "Memory maps (flutter/runner):" << std::endl;
  FILE* maps = fopen("/proc/self/maps", "r");
  if (maps) {
    char line[512];
    while (fgets(line, sizeof(line), maps)) {
      std::string s(line);
      if (s.find("flutter") != std::string::npos || s.find("libflutter") != std::string::npos) {
        std::cerr << "  " << s;
      }
    }
    fclose(maps);
  }
  
  std::cerr << "Backtrace:" << std::endl;
  
  BacktraceState state = {0};
  _Unwind_Backtrace(unwind_callback, &state);
  
  std::cerr << "=== END BACKTRACE ===" << std::endl;
  
  // Re-raise to get default behavior
  signal(sig, SIG_DFL);
  raise(sig);
}
```

**三个关键信息**：

| 字段 | 含义 | 用途 |
| ------ | ------ | ------ |
| Signal | 崩溃类型 | SIGABRT=主动 abort（如 `FML_CHECK` 失败），SIGSEGV=非法内存访问 |
| Memory maps | 各库在内存中的加载位置 | 计算 PC 地址到 ELF 文件偏移 |
| Backtrace | 崩溃时的调用栈 PC 地址 | 反解出函数名和源码行号 |

## 计算文件偏移

文件偏移 = PC地址 - 映射起始地址 + 映射文件偏移

1. 匹配 PC 落在 engine 的段（例如范围 78c548a000-78c7610000）
2. 转文件偏移 — 减去段的起始地址，加上段的文件偏移
3. ARM64 所有指令固定 4 字节宽。`bl`（branch-and-link）执行时把 **PC+4**（下一条指令地址）存入 LR 作为返回地址。backtrace unwinder 从栈上拿到的全部是这些返回地址。传给 `addr2line` 的偏移必须减 4。唯一例外是帧 #0：它来自 `_Unwind_Backtrace` → `_Unwind_GetIP()` 在信号处理器上下文的首帧 IP，不是 bl 返回地址，且 ARM64 libunwind 首帧 IP 本身不可靠（已知局限），跳过修正。

> ARM32 Thumb 模式指令 2 字节，x86 变长指令，都不能硬编码 4。本指南只针对 ARM64。

### 计算第3帧的偏移并反解

> 注意 + 0x00000000 是因为该段的文件偏移为 0

Offset = 0x78c6d925a8 - 0x78c548a000 + 0x00000000 - 4 = 0x19085a4

FML_CHECK 是宏，展开后变成 LogMessage 对象的构造、流写入、析构调用。这些展开出来的指令来自宏定义模板，DWARF 不知道把它们映射回
compositor_context.cc:119，只记录了函数名。所以函数名准、行号丢。

帧 #9 — function.h:0 — 模板实例化

Pipeline::Consume 是模板类的方法，定义在头文件 pipeline.h 里。模板在实例化时才生成代码，DWARF 只标记了 .h 文件，没标记具体行
